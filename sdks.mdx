## Quick Start

<CodeGroup>

```python Python
from apitwitter import ApiTwitter

client = ApiTwitter("tda_your_api_key")

# Get user profile
user = client.get_user("elonmusk")
print(user["data"]["name"])        # Elon Musk
print(user["data"]["followers"])   # 236446942
```

```typescript JavaScript / TypeScript
import { ApiTwitter } from "@apitwitter/sdk";

const client = new ApiTwitter("tda_your_api_key");

const user = await client.getUser("elonmusk");
console.log(user.data.name);       // Elon Musk
console.log(user.data.followers);  // 236446942
```

</CodeGroup>

### Get Followers

<CodeGroup>

```python Python

for follower in result["followers"]:
    print(f"@{follower['userName']} — {follower['name']}")
```


## Write Endpoints (POST — Your Credentials)

Write operations require your own Twitter cookies and a proxy.

```python Python — Setup
COOKIE = "ct0=YOUR_CT0;auth_token=YOUR_AUTH_TOKEN"
PROXY = "http://user:pass@host:port"
```

```typescript JavaScript — Setup
const cookie = "ct0=YOUR_CT0;auth_token=YOUR_AUTH_TOKEN";
const proxy = "http://user:pass@host:port";
```

<CodeGroup>

## Error Handling

Both SDKs throw typed exceptions for common API errors.

<CodeGroup>

```python Python
from apitwitter.exceptions import (
    ApiTwitterError,
    AuthenticationError,
    InsufficientCreditsError,
    RateLimitError,
    NotFoundError,
    ValidationError,
)

try:
    user = client.get_user("someone")
except AuthenticationError:
    print("Invalid API key")
except InsufficientCreditsError:
    print("Top up your balance")
except RateLimitError as e:
    print(f"Rate limited — retry in {e.retry_after}s")
except NotFoundError:
    print("User not found")
except ApiTwitterError as e:
    print(f"API error {e.status_code}: {e.message}")
```

```typescript JavaScript
import {
  ApiTwitterError,
  AuthenticationError,
  InsufficientCreditsError,
  RateLimitError,
  NotFoundError,
  ValidationError,
} from "@apitwitter/sdk";

try {
  const user = await client.getUser("someone");
} catch (e) {
  if (e instanceof AuthenticationError) {
    console.log("Invalid API key");
  } else if (e instanceof InsufficientCreditsError) {
    console.log("Top up your balance");
  } else if (e instanceof RateLimitError) {
    console.log(`Rate limited — retry in ${e.retryAfter}s`);
  } else if (e instanceof NotFoundError) {
    console.log("User not found");
  } else if (e instanceof ApiTwitterError) {
    console.log(`API error ${e.statusCode}: ${e.message}`);
  }
}
```

</CodeGroup>

| Exception | HTTP Code | When |
|---|---|---|
| `AuthenticationError` | 401 | Invalid or missing API key |
| `InsufficientCreditsError` | 402 | Not enough credits |
| `RateLimitError` | 429 | Too many requests (has `retry_after` / `retryAfter`) |
| `NotFoundError` | 404 | Resource not found |
| `ValidationError` | 400 | Invalid request parameters |

## Pagination Pattern

Most list endpoints support cursor-based pagination.

<CodeGroup>

```python Python
all_followers = []
cursor = None

while True:
    result = client.get_followers("username", count=200, cursor=cursor)
    all_followers.extend(result["followers"])

    cursor = result.get("next_cursor")
    if not cursor:
        break

print(f"Collected {len(all_followers)} followers")
```

```typescript JavaScript
const allFollowers: any[] = [];
let cursor: string | undefined;

do {
  const result = await client.getFollowers("username", 200, cursor);
  allFollowers.push(...result.followers);
  cursor = result.next_cursor;
} while (cursor);

console.log(`Collected ${allFollowers.length} followers`);
```

</CodeGroup>
